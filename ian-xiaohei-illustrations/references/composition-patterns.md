# Composition Patterns and Originality Rules

## Basic structure types

Choose one structure. Do not mix too many.

### Workflow

Suitable for: input -> process -> output, content production, AI workflows, automation chains.

Drawing: input on the left, Xiaohei or a strange machine processing in the middle, output on the right. Orange arrows express the main direction.

### System segment

Suitable for: information sources, filters, databases, renderers, agent system segments.

Drawing: only draw 3-5 core modules. Xiaohei participates in one key action.

### Before/after contrast

Suitable for: chaotic/orderly, manual/automated, scattered/gathered, anxious/stable.

Drawing: left is chaotic, right is stable, with an orange arrow in the middle. The character can be more exaggerated.

### Role states

Suitable for: user pain points, creator mindset, too many tools, information anxiety, stuck-to-running transitions.

Drawing: 2-4 state panels, each with one short label.

### Concept metaphor

Suitable for: content factories, information warehouses, mental black boxes, workflow machines, automated reports.

Drawing: one large strange object or machine, a few inputs, one output. It should have a memory hook.

### Layered method

Suitable for: methodology frameworks, system hierarchy, capability stacks, content system layers.

Drawing: stacked boxes, not a formal pyramid. Xiaohei is beside it building or carrying bricks.

### Route map

Suitable for: idea-to-launch, user journeys, content handoff paths, learning routes.

Drawing: one curving path and a few nodes. Xiaohei pulls a line or walks it.

### Mini comic storyboard

Suitable for: failure to success, real process, commentary, before-and-after changes.

Drawing: 2-4 small scenes, each panel expressing one action.

## Original metaphor generation method

Always invent a new metaphor from the current article. Do not copy old imagery.

### Three steps

1. Turn an abstract concept into a physical action: stuck, leaked, weighted, sorted, settled, fermented, opened, folded, unpacked, refluxed.
2. Turn a system structure into a low-tech object: broken machine, cardboard box, drawer, pipe, mailbox, strange dial, scale, well, ladder, weird station.
3. Let Xiaohei perform the action: not standing aside, but stuck in the machine, pulling the wrong wire, guarding a door, carrying, repairing, weighing, holding the ladder, taking notes, stuffing something into a weird device.

### Usable object pool

- cardboard box, drawer, old machine, funnel, scale, mailbox, door, well, ladder, pipe, ball of wires, gate, turntable, black box, hole punch, pasta press, clothesline, strange workstation.
- Use only 1-2 objects at a time. Do not overcrowd the image.

### Xiaohei action pool

- pull, carry, stuff, scoop, press, weigh, sew, cut, twist, guard, push, connect, dismantle, label, recycle.
- The action should serve the core meaning. Don't make it strange just for strangeness.

## Repetition rules

Do not automatically open or copy `assets/examples/`. These images are only for style calibration, such as line density, whitespace, color restraint, and Xiaohei attitude.

Unless the user explicitly says "copy this image / remake this composition / use this case as reference," do not directly reuse old compositions from these examples:

- two conveyor belt breakpoints
- Xiaohei pulling a decision lever inside a content machine
- Xiaohei turning into a funnel to sort flow/trust/conversion
- Xiaohei cutting a material fish
- Xiaohei pulling a handoff path
- Xiaohei pulling three layers of information sources
- three Xiaohei each holding a loudspeaker/building a bridge/opening a door
- Xiaohei stamping a messaging toolbox
- Xiaohei holding a warning sign to inspect a pit path

Even for similar themes, invent a new metaphor. For example, a "handoff path" does not have to be a route map; it can be Xiaohei attaching a content tail to a doorknob. "One fish many uses" does not have to show a fish; it can be Xiaohei pressing a paper wad into multiple shapes.
