# QA Checklist

## Must pass

- 16:9 horizontal.
- The background is clean white.
- Includes Xiaohei.
- Xiaohei performs the core action, not just decoration.
- Does not copy old case compositions; creates a new metaphor for the current article.
- The image is absurd, creative, and interesting.
- Simple and fresh. The main subject should not exceed about 60% of the canvas.
- One image should convey only one core structure.
- Chinese labels should be few, short, and readable.
- Orange is only for the main path or arrows.
- Red is only for emphasis, problems, warnings, or results.
- Blue is only for secondary notes, feedback, or system state.

## Fail signals

If any of these occur, regenerate or edit locally:

- A top-left title like "common pitfalls / workflow / system architecture / route map" appears.
- Xiaohei looks like a mascot, sticker, or cute cartoon.
- The image looks like a PPT slide, lecture material, or formal flowchart.
- There are too many elements, arrows, or nodes.
- The text becomes long blocks of explanation.
- The background has paper texture, shadows, gradients, beige, or noise.
- Real UI screenshots or tech-y interfaces appear.
- Chinese text contains serious typos or is unreadable.
- The image is too stiff and lacks an absurd metaphor.
- The composition is too similar to old examples in `assets/examples/`.

## Iteration methods

- Too ordinary: make Xiaohei the action protagonist and add one strange but coherent metaphor.
- Too complex: remove nodes, keep one action and 3-5 short labels.
- Too cute: emphasize deadpan, blank serious expression, not cute, not mascot.
- Too PPT-like: remove titles, borders, rigid grids, and too many arrows; turn it into a hand-drawn scene.
- Too similar to old cases: keep the core idea, but change the main object and Xiaohei's action.
- Typo issues: prefer local editing; if many labels are wrong, regenerate and reduce the number of labels.

## Delivery judgement

A high-quality illustration should make the viewer first feel "a bit strange," then understand the structure within one second.

If the first impression is more like a tutorial page than a white-paper absurd sketch, it is not good enough.
